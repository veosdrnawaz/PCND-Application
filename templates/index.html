About Application

Pak Currency Note Distribution

v1.0.0

This application predicts the number of Pakistani currency notes required for a user-entered amount. It implements a Scikit-learn multi-output regressor on the backend and enforces a strict mathematical rule engine to guarantee 100% correct distributions.

Technologies Used
  • Machine Learning: RandomForestRegressor, MultiOutputRegressor
  • Backend API: FastAPI, Uvicorn, Python
  • Frontend Architecture: Material Design 3, HTML5, CSS Grid & Flexbox, Vanilla JavaScript

Distribution Rules & Terms

The note distributions generated by this application follow a strict fixed algorithm hierarchy:

  1. Rs.10 Rule: Try to include at least one Rs.10 note whenever possible.
  2. Rs.20 Rule: Then include Rs.20 notes if possible.
  3. Rs.50 Rule: Then include exactly one Rs.50 note if possible.
  4. Rs.100 Rule: Then include Rs.100 notes.
  5. Rs.500 Rule: Then include Rs.500 notes.
  6. Rs.1000 Rule: Fill the remaining amount using Rs.1000 notes.

All distributions satisfy the fundamental cash equation:

1000*n1000 + 500*n500 + 100*n100 + 50*n50 + 20*n20 + 10*n10 = Amount

Predictions are adjusted in real-time by the FastAPI server to ensure the difference between the entered amount and total note value is exactly zero.
